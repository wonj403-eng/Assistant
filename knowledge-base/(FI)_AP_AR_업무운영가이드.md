# SAP FI AP/AR 업무 운영 노트

## 1. AP (Accounts Payable) 주요 업무

### 1.1 송장 처리 흐름 (PO 기반)
```
PO 생성 (ME21N)
    │
    ▼
입고 처리 (MIGO - GR)
    │
    ▼
송장 입력 (MIRO)
    │  - 3-Way Match: PO / GR / Invoice
    ▼
지급 처리 (F110 or F-53)
    │
    ▼
은행 이체 실행
```

### 1.2 주요 T-Code
| T-Code | 기능 |
|--------|------|
| MIRO | 송장 입력 (LIV) |
| MIR4 | 송장 조회 |
| MRBR | 송장 블록 해제 |
| MR8M | 송장 역전 |
| FBL1N | Vendor 명세 조회 |
| FK02 | Vendor 마스터 변경 |

## 2. AR (Accounts Receivable) 주요 업무

### 2.1 청구/수금 흐름
```
판매 오더 (VA01)
    │
    ▼
출고/납품 (VL01N)
    │
    ▼
청구서 생성 (VF01)
    │  → FI 자동 전기 (SD-FI 연동)
    ▼
AR 미결항목 관리 (FBL5N)
    │
    ▼
입금 처리 (F-28 or 자동 대사)
    │
    ▼
대사 완료 (F-32)
```

### 2.2 주요 T-Code
| T-Code | 기능 |
|--------|------|
| VF01/VF02 | 청구서 생성/변경 |
| FBL5N | Customer 명세 조회 |
| F-28 | 수동 입금 처리 |
| F-32 | AR 대사 (Clear) |
| FD32 | 여신 한도 설정 |

## 3. 지급 수단 (Payment Method) 설정

| 코드 | 방법 | 비고 |
|------|------|------|
| T | 계좌이체 | 국내 이체, 기본 사용 |
| F | 외화 송금 | 해외 지급, SWIFT |
| C | 수표 | 수기 수표 (자사 미사용) |
| B | 자동이체 | 정기 지급용 |

## 4. Dunning (독촉) 설정 운영

### 독촉 레벨 구성 (자사 기준)
| Level | 기준일수 | 방법 | 내용 |
|-------|---------|------|------|
| 1 | 5일 초과 | Email | 1차 안내 |
| 2 | 15일 초과 | Email + 우편 | 2차 독촉 |
| 3 | 30일 초과 | 팩스 | 3차 경고 |
| 4 | 45일 초과 | 법적 조치 | 법무팀 이관 |

### 독촉 실행: F150
- Run Date: 매주 월요일 배치 실행
- 담당자: FI AR팀

## 6. 자주 묻는 문제 (FAQ)

**Q: MIRO 시 "Document balance not zero" 오류**  
A: PO 금액 ↔ 송장 금액 불일치. PO 단가/수량 확인 후 허용 편차(Invoice Tolerance) 범위 검토

**Q: F110 실행 후 지급 전표가 생성되지 않음**  
A: Vendor 마스터의 Payment Method 설정, Due Date 범위, House Bank 설정 확인

**Q: FBL1N/FBL5N 에서 항목이 조회되지 않음**  
A: Posting Date 범위 및 Open Items / All Items 선택 옵션 재확인
